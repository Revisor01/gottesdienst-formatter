## Directory Purposes

**web/:**
- Purpose: Flask application and all web-related code
- Contains: Application logic, routes, templates, Docker configuration
- Key files: `app.py`, `churchdesk_api.py`, templates

**web/templates/:**
- Purpose: Jinja2 HTML templates for web interface
- Contains: All `.html` files for rendering views
- Key files: `index.html` (main), `result.html` (output display)

**.planning/codebase/:**
- Purpose: GSD codebase analysis documents
- Contains: Architecture documentation, structure maps, conventions, concerns
- Generated by: GSD mapping tools

**Project root:**
- Purpose: Standalone scripts and documentation
- Contains: Legacy CLI tools, deployment instructions, German user guides

## Key File Locations

**Entry Points:**

- `web/app.py`: Main Flask application (line 403-404) - web interface
  - Initialization: `app = Flask(__name__)`
  - Server start: `app.run(debug=True, host='0.0.0.0', port=5000)`

- `gottesdienst_formatter_final.py`: Standalone CLI tool - command-line processing
  - Interactive file input/output for Excel formatting
  - No web server required

**Configuration:**

- `web/docker-compose.yml`: Container orchestration
  - Service name: `gottesdienst-formatter`
  - Container port: 5000 → host port 5001
  - Environment: Flask production mode, ChurchDesk tokens

- `web/requirements.txt`: Python dependencies (Flask, pandas, openpyxl, requests, pytz)

- `CLAUDE.md`: Deployment instructions, server details, token management

**Core Logic:**

- `web/app.py` (lines 36-170): Formatting utility functions
  - `format_date()` - German date formatting
  - `format_time()` - German time formatting
  - `format_service_type()` - Service type abbreviations
  - `format_pastor()` - Pastor name normalization
  - `process_excel_file()` - Main Excel processing logic

- `web/churchdesk_api.py` (lines 43-542): API client and data transformation
  - `ChurchDeskAPI` - Single organization API wrapper
  - `MultiOrganizationChurchDeskAPI` - Multi-org aggregator
  - `EventAnalyzer` - Event formatting and validation
  - `extract_boyens_location()` - Location mapping logic
  - `format_boyens_pastor()` - Boyens-specific pastor formatting
  - `ORGANIZATIONS` - Configuration for 5 church organizations

**Web Routes:**

- `web/app.py` (lines 171-340):
  - `@app.route('/')` - Index page with forms
  - `@app.route('/upload', POST)` - Excel file processing
  - `@app.route('/download', POST)` - Text file download
  - `@app.route('/fetch_churchdesk_events', POST)` - ChurchDesk event fetching
  - `@app.route('/export_selected_events', POST)` - Event export to Boyens format

**Testing:**

- No formal test files in repository
- Manual testing with sample Excel files
- ChurchDesk API tested via web form

## Naming Conventions

**Files:**

- Python modules: `snake_case.py` (e.g., `churchdesk_api.py`, `app.py`)
- Templates: `lowercase.html` (e.g., `base.html`, `result.html`)
- Standalone scripts: `descriptive_snake_case_final.py` (e.g., `gottesdienst_formatter_final.py`)
- Documentation: UPPERCASE.md (e.g., `CLAUDE.md`, `README.md`)

**Directories:**

- Web app: `web/` (singular, lowercase)
- Template directory: `templates/` (plural, lowercase)
- Analysis: `.planning/codebase/` (hidden dot directories)

**Python Functions:**

- Formatting: `format_X()` pattern (e.g., `format_date`, `format_pastor`, `format_service_type`)
- Classes: PascalCase (e.g., `ChurchDeskAPI`, `EventAnalyzer`, `MultiOrganizationChurchDeskAPI`)
- Variables: `snake_case` (e.g., `formatted_text`, `selected_orgs`, `events_by_date`)
- Constants: `UPPERCASE` (e.g., `ORGANIZATIONS`, `LOCATION_MAPPINGS`, `MULTI_CHURCH_CITIES`)

**Database/Data:**

- No persistent database; uses temporary files
- Excel columns: German titles (e.g., `Startdatum`, `Titel`, `Standortnamen`, `Mitwirkender`)
- JSON keys in templates: Camel case (e.g., `startDate`, `organization_id`)

## Where to Add New Code

**New Feature (Format Type or Location):**
- Primary code: `web/churchdesk_api.py` (lines 320-355 for location mappings or lines 358-422 for pastor formatting)
- Secondary: `web/app.py` (lines 71-101 for service type formatting)
- Tests: Manual testing with representative Excel/API data

**New Web Route:**
- Location: `web/app.py` after line 339
- Pattern: Use `@app.route()` decorator with methods list
- Add corresponding template in `web/templates/` if needed
- Return `render_template()` or `send_file()`

**New Template:**
- Location: `web/templates/`
- Pattern: Extend `base.html` with `{% extends "base.html" %}`
- Style: Use Bootstrap 5.1.3 classes (defined in base.html)
- Colors: Use CSS variables `--primary-color`, `--primary-dark`, `--light-bg`

**New Organization in ChurchDesk:**
- Location: `web/churchdesk_api.py` lines 15-41 (ORGANIZATIONS dict)
- Add entry: `org_id: {'name': 'Display Name', 'token': 'api_token_here', 'description': 'Details'}`
- Update: `web/templates/index.html` to add checkbox for new organization

**Utility Function:**
- Simple formatting: Add to `web/app.py` with other format functions
- Complex data transformation: Add to `web/churchdesk_api.py` as method or helper function
- Both: Consider making it a Jinja2 filter (see `format_parish_name` pattern at app.py line 21)

**Excel Processing Enhancement:**
- Modify: `process_excel_file()` function in `web/app.py` lines 127-169
- Add column validation if needed
- Update `required_columns` list at line 133
