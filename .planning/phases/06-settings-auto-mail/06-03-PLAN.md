**web/mail_service.py** — Neues Modul mit zwei Hauptfunktionen:

1. **`generate_next_month_export() -> str`**:
   - Berechnet naechsten Monat: wenn heute Maerz → April. Jahresuebergang beachten (Dezember → Januar naechstes Jahr).
   - Alle ORGANIZATIONS aus config.py nutzen (alle Org-IDs).
   - `create_multi_org_client(list(ORGANIZATIONS.keys()))` aufrufen.
   - `client.get_monthly_events(year, month, gottesdienst_only=True)` fuer den Folgemonat.
   - Events durch `EventAnalyzer.format_event_for_boyens()` filtern und formatieren.
   - `convert_churchdesk_events_to_boyens(processed_events)` aufrufen.
   - Return: formatierter Text als String.
   - Bei leerem Ergebnis: Return leerer String (kein Fehler).

2. **`send_boyens_mail(user_settings, export_text, secret_key) -> tuple[bool, str]`**:
   - Parameter: UserSettings-Objekt, formatierter Text, SECRET_KEY fuer Entschluesselung.
   - SMTP-Passwort entschluesseln: `decrypt_value(user_settings.smtp_password_encrypted, secret_key)`.
   - Email zusammenbauen mit `email.mime.multipart.MIMEMultipart` (mixed):
     - From: user_settings.sender_email
     - To: user_settings.recipient_email
     - Subject: "Gottesdienste {Monat} {Jahr} — Boyens-Export" (Folgemonat)
     - Body (MIMEText, plain): Der formatierte Text direkt im Body.
     - Attachment (MIMEText, plain): Gleicher Text als .txt-Anhang. Filename: `gottesdienste_{monat}_{jahr}.txt`. Content-Disposition: attachment.
   - Versand via `smtplib.SMTP(server, port, timeout=30)`:
     - `ehlo()`, `starttls()`, `login(username, password)`, `send_message(msg)`, `quit()`
   - Return: `(True, 'Erfolgreich gesendet')` oder `(False, 'Fehler: {str(e)}')`
   - Exception-Handling: smtplib.SMTPAuthenticationError → "SMTP-Authentifizierung fehlgeschlagen", smtplib.SMTPException → allgemeine Meldung, socket.timeout → "Verbindung zum SMTP-Server abgelaufen", Exception → generischer Fehler.

3. **`get_effective_send_date(year, month, day) -> datetime.date`**:
   - Berechnet den effektiven Versandtag unter Beruecksichtigung der Wochenend-Logik.
   - Wenn day > letzter Tag des Monats: letzter Tag verwenden (z.B. Tag 28 im Februar ist OK per Constraint 1-28).
   - `datetime.date(year, month, day)` erstellen.
   - Wenn Samstag (weekday()==5): 1 Tag zurueck (Freitag).
   - Wenn Sonntag (weekday()==6): 2 Tage zurueck (Freitag).
   - Return: date-Objekt.

Imports: smtplib, email.mime.text, email.mime.multipart, email.mime.base, datetime, logging. Logging mit `logger = logging.getLogger(__name__)`.

1. **web/scheduler.py** — APScheduler-Modul:

   **`init_scheduler(app)`**:
   - Import: `from apscheduler.schedulers.background import BackgroundScheduler`
   - Scheduler nur starten wenn NICHT im Test-Modus (`app.config.get('TESTING')` ist False) und NICHT im Flask-CLI (`os.environ.get('FLASK_RUN_FROM_CLI')` oder Werkzeug-Reloader: nur starten wenn `os.environ.get('WERKZEUG_RUN_MAIN') == 'true'` ODER kein WERKZEUG_RUN_MAIN gesetzt (= Gunicorn/Produktion)). Einfacher Check:
     ```python
     if app.config.get('TESTING'):
         return
     # Werkzeug-Reloader: nur im Main-Prozess starten
     if os.environ.get('WERKZEUG_RUN_MAIN') == 'false':
         return
     ```
   - `scheduler = BackgroundScheduler(daemon=True)`
   - CronTrigger: taeglich um 8:00 Uhr → `scheduler.add_job(check_and_send_mails, 'cron', hour=8, minute=0, args=[app], id='daily_mail_check', replace_existing=True)`
   - `scheduler.start()`
   - `app._scheduler = scheduler` (fuer Shutdown-Zugriff)
   - atexit-Handler: `import atexit; atexit.register(lambda: scheduler.shutdown(wait=False))`

   **`check_and_send_mails(app)`**:
   - Innerhalb von `with app.app_context():`
   - Heute: `datetime.date.today()`
   - Alle UserSettings mit `auto_send_enabled=True` und nicht-leerem smtp_server laden.
   - Fuer jeden User:
     - `effective_date = get_effective_send_date(today.year, today.month, settings.send_day)`
     - Wenn `effective_date == today`:
       - Pruefen ob bereits diesen Monat gesendet: `settings.last_send_date` im selben Monat → skip.
       - `export_text = generate_next_month_export()`
       - Wenn export_text leer: `last_send_status = 'Keine Events fuer naechsten Monat'`, skip.
       - `success, message = send_boyens_mail(settings, export_text, app.config['SECRET_KEY'])`
       - Wenn success: `settings.last_send_date = datetime.now(timezone.utc)`, `settings.last_send_status = message`
       - Wenn failure: Retry nach 1 Stunde einplanen:
         ```python
         scheduler = app._scheduler
         retry_time = datetime.now() + timedelta(hours=1)
         scheduler.add_job(
             retry_send_mail, 'date', run_date=retry_time,
             args=[app, settings.id, export_text],
             id='retry_mail_{}'.format(settings.user_id),
             replace_existing=True
         )
         ```
       - `db.session.commit()`

   **`retry_send_mail(app, settings_id, export_text)`**:
   - Innerhalb von `with app.app_context():`
   - UserSettings by id laden
   - `success, message = send_boyens_mail(settings, export_text, app.config['SECRET_KEY'])`
   - `settings.last_send_date = datetime.now(timezone.utc)` (auch bei Fehler — markiert als "versucht")
   - `settings.last_send_status = message` (bei Fehler: "Fehlgeschlagen nach Retry: {message}")
   - `db.session.commit()`
   - Logging: `logger.info(...)` bei Erfolg, `logger.error(...)` bei Fehler.

   Imports: os, logging, datetime, atexit, apscheduler, models, mail_service, crypto, extensions.
